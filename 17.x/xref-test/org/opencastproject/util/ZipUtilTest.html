41  /* WARNING
42   *
43   * Instead of using a single static final file for testing the destination files, we are using
44   * a file with a unique name for each test. The reason is that if the tests are run so quickly
45   * the filesystem is unable to erase the files on time and a test can re-create the file before
46   * it was physically deleted. If those files are suppossed to be different, a test may return
47   * a false positive or negative. If the pathnames are different for each test, there's no
48   * possible mistake.
49   * For uniformity, all the tests declare internally it's own destination file/filename, even
50   * though this may not be strictly necessary for all of them.
51   *
52   * ruben.perez
53   */
